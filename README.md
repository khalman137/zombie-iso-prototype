# Новый движок 2.5D — тест архитектуры v1.1

Сборка не продолжает старую архитектуру.

Что реализовано:
- чанк 64×64;
- отсутствующий чанк считается травой;
- карта без встроенных домов;
- building_spawns + building_facing;
- выбор шаблона без вращения;
- отдельные ZIP-шаблоны участков;
- floor как назначение помещения;
- walls 131×131 с внешней рамкой;
- автоматическое разделение дома и гаража по связности пола z0;
- независимое скрытие крыши текущего здания;
- отображение этажей 0...Z внутри текущего здания;
- скрытие этажей выше игрока только у текущего здания;
- лестницы длиной 3 клетки из zN_stairs.png;
- переход вверх/вниз только при нахождении на лестнице;
- запрет выхода за границы пола на верхних этажах;
- жёсткая палитра PNG.

Управление:
- WASD/стрелки или левый виртуальный стик — движение;
- E или кнопка «Этаж +» — подняться;
- Q или кнопка «Этаж −» — спуститься.

Переход работает только на фиолетовых клетках лестницы и только если лестница существует на обоих связанных этажах.

Запуск: разместить содержимое папки на HTTP/HTTPS-сервере и открыть index.html. Из-за fetch ZIP-файлов прямой запуск через file:// может быть заблокирован браузером.

## v1.2 — плавные лестницы

- Подъём и спуск выполняются обычным движением по трём клеткам лестницы.
- Высота персонажа изменяется непрерывно от z0 до z1.
- Активный этаж переключается на середине второй клетки лестницы (50% подъёма).
- Во время движения по лестнице персонаж удерживается вдоль её оси.
- Кнопки ручного переключения этажей удалены.


## Архивная структура v1.3

Все каталоги данных заменены архивами для прямой загрузки в GitHub:

- `maps.zip` — содержит отдельные ZIP-архивы карт;
- `templates.zip` — содержит отдельные ZIP-архивы участков;
- `docs.zip` — документация формата.

Движок читает вложенные архивы напрямую через JSZip. Распаковывать `maps.zip` и `templates.zip` не требуется. Для добавления новой карты или шаблона нужно добавить его ZIP внутрь соответствующего пакета и обновить индекс/manifest, когда он будет введён.
